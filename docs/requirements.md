# AI集客OS — 要件定義書

> バージョン: 1.0
> 作成日: 2026-03-10
> 対象システム: gyosei-os（coreai-x.com）

---

## 1. プロダクト概要・背景・目的

### 1.1 プロダクト名

**AI集客OS**

### 1.2 コンセプト

行政書士事務所の集客Webサイトを、AIが5分以内に自動生成するSaaSプラットフォーム。
質問ウィザードに答えるだけで、本番品質のWebサイトが即座に生成・公開される。

### 1.3 背景

- 行政書士は全国に5万人超存在するが、大半の事務所は費用・時間・技術的ハードルからWebサイトを持てていない
- 従来の制作会社への依頼は50〜100万円・数か月を要し、士業にとって現実的でない
- 生成AIの進化により、プロ品質のコピーライティングを自動化できる環境が整った

### 1.4 目的

1. Web制作の専門知識ゼロでも5分で集客サイトを公開できる体験を提供する
2. リード（問い合わせ）獲得から管理までをワンプラットフォームで完結させる
3. 将来的に他士業・飲食店・整体院等へ業種を拡張するための共通基盤を構築する

## 3. ユーザーストーリー（主要フロー）

### 3.1 初回サイト生成フロー（コアジャーニー）

```
[行政書士] がランディングページ（coreai-x.com）を訪問する
    ↓
「5分で無料作成」ボタンをクリックし /onboard へ遷移する
    ↓
9ステップの質問ウィザードを回答する
    - 事務所名、代表者名、メールアドレス
    - 代表者経歴、業務内容、対応エリア
    - 強み、文章スタイル、お客様の声（任意）
    ↓
Claude API がサイトコンテンツを自動生成する（約30秒）
    ↓
プレビュー画面（/onboard/preview/[slug]）でサイトを確認する
    ↓
テキストをクリックして気になる箇所を直接編集する
    ↓
「サイトを公開する」→ プラン選択 → 名前・メールを登録する
    ↓
サイトが {slug}.coreai-x.com で公開される
```

### 3.2 ログイン・ダッシュボードフロー

```
[登録済みユーザー] が app.coreai-x.com/login を開く
    ↓
メールアドレスを入力する
    ↓
マジックリンクメールが届く（15分有効）
    ↓
メール内のリンクをクリックする → /dashboard/[slug] にリダイレクトされる
    ↓
ダッシュボードで以下を管理する
    - サイトの編集・公開/非公開の切替
    - 問い合わせ（リード）の確認
    - SEOページの管理
    - 広告管理
```

### 3.3 問い合わせ受信フロー

```
[エンドユーザー] が {slug}.coreai-x.com を訪問する
    ↓
お問い合わせフォームに名前・メール・電話・メッセージを入力して送信する
    ↓
[行政書士] にリード通知メールが届く（Resend 経由）
    ↓
[エンドユーザー] に自動返信メールが届く
    ↓
[行政書士] が /dashboard/[slug]/leads でリードを管理する
```

### 3.4 プロ確認プラン申込フロー

```
[行政書士] がプレビュー画面で「プロに確認してもらう（¥100,000）」を選択する
    ↓
レビュアーを選択する（個人・ダブルチェック）
    ↓
名前・メールを登録する → ReviewRequest レコードが作成される（status: pending）
    ↓
[オーナー] が admin.coreai-x.com/reviews で依頼を確認する
    ↓
レビュアーを担当に割り当て、技術確認後に承認する（status: approved）
    ↓
[行政書士] に承認通知メールが届く
```

## 5. 非機能要件

### 5.1 性能要件

| 項目 | 要件 |
|------|------|
| サイト生成時間 | AI生成含め60秒以内（Vercel maxDuration=60） |
| 公開サイトのページロード | First Contentful Paint 2秒以内（LCP 2.5秒以内） |
| 問い合わせフォーム送信 | レスポンス3秒以内 |
| ダッシュボード画面遷移 | 1秒以内 |
| DB接続 | Neon サーバーレスプール接続、コネクションプーリング有効 |
| AIコスト最適化 | 同一入力の SHA-256 ハッシュキャッシュにより重複AI呼び出しを防止 |

### 5.2 セキュリティ要件

| 項目 | 要件 |
|------|------|
| 認証 | ユーザー: HMAC-SHA256 JWT（Web Crypto API、Edge Runtime対応）/ 管理者: NextAuth.js + bcrypt |
| マジックリンク | 有効期間15分、使用後即時無効化 |
| セッションCookie | httpOnly=true、secure=true（本番）、sameSite=lax |
| 管理画面アクセス制御 | admin.coreai-x.com / /admin ルートは管理者のみアクセス可 |
| ユーザーデータアクセス制御 | /dashboard/[slug] は当該サイトのオーナーのみアクセス可（セッション検証） |
| 環境変数管理 | ANTHROPIC_API_KEY・RESEND_API_KEY・AUTH_SECRET は環境変数で管理、コードにハードコードしない |
| SQLインジェクション対策 | Prisma ORM の型安全クエリを使用（生のSQL文字列は使用しない） |
| CORS | Next.js Route Handler のデフォルト設定に準拠 |

### 5.3 可用性・運用要件

| 項目 | 要件 |
|------|------|
| 稼働率目標 | 99.9%以上（Vercel SLA に依存） |
| バックアップ | Neon 自動バックアップ（日次） |
| デプロイ | Vercel への git push で自動デプロイ（ゼロダウンタイム） |
| モニタリング | Vercel Function Logs / エラーは console.error でログ出力 |
| スケーラビリティ | サーバーレスアーキテクチャによりトラフィック急増時も自動スケール |

### 5.4 ユーザビリティ要件

| 項目 | 要件 |
|------|------|
| レスポンシブ対応 | PC・スマートフォン・タブレット対応（モバイルファースト設計） |
| ブラウザ対応 | Chrome・Safari・Firefox・Edge の最新版 |
| アクセシビリティ | 基本的なセマンティックHTML・alt属性の使用 |
| 言語 | 日本語のみ（多言語対応は将来課題） |

### 5.5 拡張性要件

| 項目 | 要件 |
|------|------|
| 業種拡張 | 業種はDBの属性で管理し、ドメイン構成は変えずに対応可能 |
| サブドメイン | ワイルドカードサブドメイン（*.coreai-x.com）で任意スラッグに対応 |
| AIモデル更新 | generator.ts の model 定数を変更するだけで切替可能 |
