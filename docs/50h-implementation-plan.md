# 50時間実装計画

## 前提

- 実装者: 自分 + Claude Code
- DB: 7テーブル固定（clients / practice_areas / landing_pages / leads / invoices / ai_generation_logs / monthly_reports）
- ゴール: **自分が実際に使える状態**（クライアントを1社オンボードし、LPを公開し、問い合わせを受け、レポートを送れる）

---

## 技術スタック（選定理由付き）

| レイヤー | 採用 | 理由 |
|---------|------|------|
| フレームワーク | Next.js 14 App Router | API・管理画面・公開LPを1リポジトリで完結 |
| DB | Neon PostgreSQL | サーバーレス、無料枠あり、Prismaとの相性◎ |
| ORM | Prisma | スキーマ定義→型安全なクエリまで速い |
| AI | Claude API (claude-sonnet-4-6) | LP生成・レポート要約の両方をカバー |
| 認証 | NextAuth.js (Credentials) | 自分だけが使う管理画面。OAuth不要 |
| スタイル | Tailwind CSS | 管理画面は速度優先。LPは別途カスタム |
| PDF | Puppeteer (Vercel対外) または react-pdf | 月次レポートPDF生成 |
| メール | Resend | 開発者体験が良い。無料枠3000通/月 |
| デプロイ | Vercel | Next.jsとの親和性。プレビューURL即時 |

---

## フェーズ別実装計画

### Phase 1: 基盤構築（10時間）

**目標**: DBが動き、管理画面にログインできる

| タスク | 時間 | 内容 |
|--------|------|------|
| プロジェクト初期化 | 1h | Next.js + Prisma + Neon接続 |
| Prismaスキーマ定義 | 2h | 7テーブル全定義 + migrate |
| 管理画面ログイン | 2h | NextAuth Credentials（固定ユーザー1人） |
| 管理画面レイアウト | 2h | サイドバー + ナビゲーション |
| クライアントCRUD | 3h | 一覧・登録・編集・詳細 |

**Phase 1終了時にできること:**
- ブラウザで管理画面を開ける
- クライアント（行政書士事務所）を登録・編集できる

---

### Phase 2: LP生成エンジン（15時間）

**目標**: AIがLPを生成し、公開URLにアクセスできる

| タスク | 時間 | 内容 |
|--------|------|------|
| LP生成フォーム | 2h | client + practice_area選択 → 生成指示入力 |
| Claude API呼び出し | 3h | プロンプト設計 + content JSONB生成 + ai_generation_logs保存 |
| LP管理画面 | 2h | 一覧・プレビュー・ステータス変更 |
| LP編集画面 | 3h | content JSONBの各フィールドを編集できるフォーム |
| 公開LPページ | 5h | `/lp/[client_slug]/[practice_area_slug]` でLP表示（ヘッドライン・FAQ・CTA・フォーム） |

**Phase 2終了時にできること:**
- クライアントを選んでAIにLP生成を指示できる
- 生成されたLPを確認・編集できる
- URLを公開してブラウザで閲覧できる

**削った機能:**
- ~~リッチなビジュアルエディタ~~ → フォームで直接テキスト編集
- ~~セクション並び替えUI~~ → 固定セクション構成
- ~~デザインテーマ切り替え~~ → 1テンプレートのみ

---

### Phase 3: リード受付（8時間）

**目標**: LPから問い合わせが入り、管理画面で確認できる

| タスク | 時間 | 内容 |
|--------|------|------|
| LP内問い合わせフォーム | 2h | 名前・電話・メール・メッセージ |
| リード保存API | 2h | UTM / referrer / source_keyword の自動取得 |
| クライアントへのメール通知 | 2h | Resendで即時通知（テンプレート固定） |
| リード管理画面 | 2h | 一覧・フィルター・ステータス変更 |

**Phase 3終了時にできること:**
- LPから問い合わせが入ると自動でDBに保存される
- クライアントのメールに通知が飛ぶ
- 管理画面でリード一覧を確認できる

**削った機能:**
- ~~リード詳細コメント~~ → メモ欄のみ
- ~~ステータス変更履歴~~ → 現在値のみ
- ~~クライアントへのPortal~~ → メール通知のみ

---

### Phase 4: 月次レポート（12時間）

**目標**: ワンクリックでレポートPDFを生成・送信できる

| タスク | 時間 | 内容 |
|--------|------|------|
| 月次集計ロジック | 3h | leads集計 → stats JSONB生成（業務別・キーワード別・前月比） |
| AIレポート要約生成 | 2h | Claude API → ai_summary + ai_suggestions |
| レポート確認・編集画面 | 3h | 集計データ表示 + テキスト編集 + ステータス管理 |
| PDF生成 | 3h | react-pdf でレポートPDF出力 |
| メール送信 | 1h | Resend で PDF添付送信 |

**Phase 4終了時にできること:**
- 月初ボタン1押しで集計＋AI要約が生成される
- 管理画面でテキストを修正できる
- ボタンでPDF生成→メール送信できる

**削った機能:**
- ~~cron自動実行~~ → 手動ボタンで月初に実行（Vercel Cron は後で追加）
- ~~グラフ・チャート~~ → テキストと数値のみ
- ~~高デザインなPDF~~ → シンプルな構成

---

### Phase 5: 請求管理（5時間）

**目標**: 請求書を発行して状態を管理できる

| タスク | 時間 | 内容 |
|--------|------|------|
| 請求CRUD | 2h | 一覧・登録・ステータス変更 |
| 請求書PDF生成 | 2h | react-pdf で請求書フォーマット |
| 入金確認フロー | 1h | paid_at 記録・ステータス更新 |

**削った機能:**
- ~~Stripe連携~~ → 手動入金確認
- ~~自動請求生成~~ → 手動で毎月作成

## 50時間後にできること / できないこと

### ✅ できること

- クライアントを登録してAIにLPを生成させる
- LPを公開URLで公開する（`/lp/[client_slug]/[practice_area_slug]`）
- LPのフォームから問い合わせを受け取る
- 問い合わせをクライアントにメール通知する
- 月次でリードを集計してAI要約を生成する
- レポートを確認・修正してPDFで送付する
- 請求書を発行して入金管理する

### ❌ 50時間内には作らないもの

| 機能 | 理由 |
|------|------|
| LP ビジュアルエディタ | 工数大。JSONフォームで代替 |
| LP デザイン複数テーマ | 1テンプレートで十分 |
| GA連携・アクセス解析 | GAのスクリプト埋め込みのみ |
| cron自動実行 | 手動ボタンで代替（後から5分で追加可） |
| Stripe請求連携 | 手動で十分 |
| クライアントPortal | メール通知で代替 |
| A/Bテスト | 後フェーズ |

---

## 実装順序の根拠

```
クライアント登録 → LP生成 → LP公開 → リード受付
     ↓
 これだけで「使える状態」

     ↓ 追加
月次レポート → 請求管理
     ↓
 これで「売れる状態」
```

Phase 3 完了時点（33時間）で実際のクライアントに使い始められる。
Phase 4, 5 は並行して実務しながら実装可能。

---

## 最初の1時間でやること

```bash
npx create-next-app@latest laratter --typescript --tailwind --app
cd laratter
npm install prisma @prisma/client
npm install next-auth
npm install @anthropic-ai/sdk
npm install resend
npx prisma init
```

→ Prismaスキーマを7テーブル書いて `npx prisma db push`
→ これで全テーブルが立ち上がる
